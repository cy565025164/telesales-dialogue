# 流程节点指引 — get_flow_guidance 返回内容

Agent 调用 `get_flow_guidance(current_node, situation)` 时，返回对应节点的详细指引。

## opening — 开场白

**目标**: 30秒内建立对话意愿
**指引**:
- 自报家门 + 一句话说明来意
- 用客户能关联的信息破冰（行业/场景）
- 不要问"您有时间吗"（给拒绝机会）
**推进**: → needs_discovery（自然过渡到提问）

## needs_discovery — 需求挖掘

**目标**: 找到客户的真实痛点
**指引**:
- 用开放式问题："您目前XX方面最大的挑战是？"
- 听到痛点后追问细节："具体是哪个环节比较头疼？"
- 不要急于介绍产品，先让客户多说
**推进**: → product_pitch（痛点明确后，针对性介绍）

## product_pitch — 产品介绍

**目标**: 将产品价值与客户痛点对接
**指引**:
- 先回应痛点，再说产品怎么解决
- 讲利益不讲参数："帮您每月省下XX小时"
- 用案例佐证："跟您类似的XX公司用了之后..."
- 不确定的信息调用 search_kb 确认
**推进**: → trial_close（出现购买信号时）| → objection_handling（出现异议时）

## objection_handling — 异议处理

**目标**: 化解顾虑，重建信心
**指引**:
- 价格异议 → 换算日均成本、强调ROI、对比隐性成本
- 需求异议 → 重新关联痛点、举反面案例
- 时间异议 → 强调限时优惠/名额有限、降低决策门槛
- 信任异议 → 提供试用/保障、引用成功案例
- **先说"理解"，再转化**，不要硬怼
**推进**: → trial_close（异议化解后）| → graceful_exit（3次无效）

## trial_close — 试探成交

**目标**: 推动决策
**指引**:
- 假设成交法："那我帮您安排一下？"
- 二选一法："您看选A套餐还是B套餐？"
- 总结价值后直接推进，不要犹豫
**推进**: → closing（客户同意）| → objection_handling（新异议）

## closing — 成交确认

**目标**: 锁定订单细节
**指引**:
- 确认关键信息（套餐/数量/联系方式）
- 说明下一步流程
- 表达感谢，不要过度兴奋
**推进**: 对话结束

## graceful_exit — 礼貌退出

**目标**: 保持好印象，留后路
**指引**:
- 不纠缠："完全理解，不勉强您"
- 留触点："加个微信，有需要随时联系"
- 真诚告别，不要阴阳怪气
**推进**: 对话结束
